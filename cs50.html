Notes

    1. Printing and Data Types

    • Use printf for formatted output, with placeholders like %i (integer), %f (float), and %li (long integer).
    • Specify precision for floats using %.nf (e.g., %.5f for five decimal places).
    • Use proper data types (e.g., int, float, double) to match the desired operation. Use float or double for precise calculations.

2. Loops and Control Structures

  • While Loop: Repeats until a condition is false; ideal for validating user input.
  • Do-While Loop: Ensures the loop runs at least once before checking the condition.
  • For Loop: Used for iteration with defined start, condition, and increment steps.
  • Loops can be nested for multi-dimensional tasks, such as printing grids dynamically based on user input.

3. Functions and Modular Design

  • Functions can return values (e.g., int, float) and take arguments for dynamic behavior.
  • Use function prototypes to declare functions before they are defined.
  • Type casting allows converting variables (e.g., int to float) to handle truncation in operations like division.
  • Abstract repetitive tasks into functions for better readability and reusability.

4. Constants and Variable Scope

  • Use const to declare variables that shouldn’t be modified.
  • Understand variable scope: A variable is only accessible within the block (curly braces) where it’s declared.

5. Arithmetic and Memory Limitations

  • Operators like +, -, *, /, and % perform basic calculations, with % used for remainders.
  • Integer Overflow: Finite memory causes numbers to wrap around (e.g., 32-bit integers max out at ~4 billion). Examples:
    • Y2K Problem: Two-digit years confused software after 1999.
    • 2038 Problem: 32-bit time counters will overflow, causing potential failures.
  • Floating Point Imprecision: Limited memory means computers cannot perfectly represent all real numbers, leading to rounding errors.

6. Efficient Design and Readability

  • Avoid redundancy using loops and dynamic variables.
  • Add comments (//) to explain code for better collaboration.
  • Modularize code into reusable functions to simplify logic and avoid duplication.

7. Real-World Programming Challenges

  • Integer Overflow caused glitches in:
    • Games like Pac-Man and Donkey Kong.
    • Aircraft software (e.g., Boeing 787 required periodic restarts).

8. Command-Line Interface (CLI)

  • Learn CLI commands like mv (rename), rm (remove), ls (list files), and mkdir (make directory) for efficient file management.
  • Autocomplete (Tab) and command history (arrow keys) streamline workflows.
